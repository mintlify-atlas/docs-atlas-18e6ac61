---

The Book My Show API is backed by a set of JPA entities that map directly to database tables. All entities extend `BaseModel`, which provides an auto-generated integer primary key. The entities are grouped below by domain: core venue infrastructure, show scheduling, and booking.

## BaseModel

Every entity inherits the following field.

| Field | Type | Description |
|-------|------|-------------|
| `id` | `integer` | Primary key. Auto-generated by the database (`IDENTITY` strategy). |

  <Accordion title="Core domain — City, Theatre, Screen, Seat" defaultOpen={true}>

    ### City

    Represents a geographical city that contains one or more theatres.

    | Field | Type | Description |
    |-------|------|-------------|
    | `id` | `integer` | Inherited from `BaseModel`. |
    | `name` | `string` | Display name of the city (e.g. `"Mumbai"`). |
    | `theatres` | `Theatre[]` | List of theatres located in this city. |

    ---

    ### Theatre

    A cinema venue located in a city, containing one or more screens.

    | Field | Type | Description |
    |-------|------|-------------|
    | `id` | `integer` | Inherited from `BaseModel`. |
    | `name` | `string` | Display name of the theatre (e.g. `"PVR Juhu"`). |
    | `address` | `string` | Street address of the theatre. |
    | `screens` | `Screen[]` | List of screens inside this theatre. |
    | `city` | `City` | The city in which this theatre is located. |

    ---

    ### Screen

    An auditorium inside a theatre. A screen hosts shows and has a fixed set of seats.

    | Field | Type | Description |
    |-------|------|-------------|
    | `id` | `integer` | Inherited from `BaseModel`. |
    | `name` | `string` | Screen identifier within the theatre (e.g. `"Screen 1"`). |
    | `seats` | `Seat[]` | All physical seats in this screen. |
    | `status` | `ScreenStatus` | Operational state. See [ScreenStatus](/api/enums#screenstatus). |
    | `features` | `Feature[]` | Presentation capabilities (e.g. `IMAX`, `DOLBY_ATMOS`). See [Feature](/api/enums#feature). |
    | `theatre` | `Theatre` | The theatre this screen belongs to. |

    ---

    ### Seat

    A single physical seat within a screen.

    | Field | Type | Description |
    |-------|------|-------------|
    | `id` | `integer` | Inherited from `BaseModel`. |
    | `name` | `string` | Row/column label (e.g. `"D4"`). |
    | `seatType` | `SeatType` | Category of the seat (e.g. `GOLD`). Determines pricing via `SeatTypeShow`. See [SeatType](/api/enums#seattype). |
    | `screen` | `Screen` | The screen this seat belongs to. |

  </Accordion>

  <Accordion title="Show and scheduling — Show, ShowSeat, SeatTypeShow, Movie">

    ### Movie

    A film title available for screening.

    | Field | Type | Description |
    |-------|------|-------------|
    | `id` | `integer` | Inherited from `BaseModel`. |
    | `name` | `string` | Title of the movie. |
    | `description` | `string` | Synopsis or marketing description. |

    ---

    ### Show

    A scheduled screening of a movie on a particular screen.

    | Field | Type | Description |
    |-------|------|-------------|
    | `id` | `integer` | Inherited from `BaseModel`. |
    | `startTime` | `Date` | When the show begins. |
    | `endTime` | `Date` | When the show ends. |
    | `features` | `Feature[]` | Presentation features for this specific show (may differ from screen capabilities). See [Feature](/api/enums#feature). |
    | `screen` | `Screen` | The screen on which this show plays. |

    ---

    ### ShowSeat

    The junction between a `Show` and a `Seat`. Tracks the real-time booking status of each seat for a given show. These are the IDs supplied in `showSeatIds` when calling `POST /tickets/book`.

    | Field | Type | Description |
    |-------|------|-------------|
    | `id` | `integer` | Inherited from `BaseModel`. |
    | `show` | `Show` | The show this record belongs to. |
    | `seat` | `Seat` | The physical seat this record represents. |
    | `status` | `SeatStatus` | Current availability of this seat for this show. See [SeatStatus](/api/enums#seatstatus). |

    ---

    ### SeatTypeShow

    Stores the price for each seat type within a specific show. Used to look up how much a `SILVER`, `GOLD`, `PLATINUM`, or `RECLINER` seat costs for a given show.

    | Field | Type | Description |
    |-------|------|-------------|
    | `id` | `integer` | Inherited from `BaseModel`. |
    | `show` | `Show` | The show these prices apply to. |
    | `seatType` | `SeatType` | The seat category (e.g. `PLATINUM`). |
    | `price` | `double` | Price in the local currency for this seat type and show combination. |

  </Accordion>

  <Accordion title="Booking — User, Ticket">

    ### User

    A registered user of the platform who can book tickets.

    | Field | Type | Description |
    |-------|------|-------------|
    | `id` | `integer` | Inherited from `BaseModel`. |
    | `name` | `string` | Full name of the user. |
    | `email` | `string` | Email address. Used for login and notifications. |
    | `password` | `string` | User's password credential. |

    ---

    ### Ticket

    A booking record created when a user successfully reserves seats for a show. Immediately after creation the status is `UNPAID` and transitions to `BOOKED` once payment is confirmed.

    | Field | Type | Description |
    |-------|------|-------------|
    | `id` | `integer` | Inherited from `BaseModel`. |
    | `show` | `Show` | The show for which this ticket was booked. |
    | `seats` | `Seat[]` | The physical seats reserved by this ticket. |
    | `timeOfBooking` | `Date` | Timestamp of when the ticket was created. |
    | `user` | `User` | The user who booked the ticket. |
    | `status` | `TicketStatus` | Lifecycle state of the ticket (`UNPAID`, `BOOKED`, or `CANCELLED`). See [TicketStatus](/api/enums#ticketstatus). |

  </Accordion>
