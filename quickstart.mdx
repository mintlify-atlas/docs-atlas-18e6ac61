This guide walks you through getting the Book My Show Ticket Booking API running on your machine and making your first booking request. The whole process takes a few minutes.

## Prerequisites

- **Java 17 or later** — check with `java -version`
- **Maven 3.x** — check with `mvn -version`

## Steps

<Steps>
  <Step title="Clone the repository">
    Clone the source code from GitHub.

    ```bash
    git clone https://github.com/Siddarth1005/book_my_show_2.git
    cd book_my_show_2
    ```
  </Step>

  <Step title="Build with Maven">
    Compile the project and download dependencies.

    ```bash
    mvn clean install
    ```

    Maven downloads all dependencies declared in `pom.xml` and compiles the application. A `BUILD SUCCESS` message confirms the build completed without errors.
  </Step>

  <Step title="Run the server">
    Start the Spring Boot application.

    ```bash
    mvn spring-boot:run
    ```

    The server starts on port **8080** by default. You will see log output ending with a line similar to:

    ```
    Started BmsBookTicketApplication in X.XXX seconds
    ```
  </Step>

  <Step title="Verify the server is up">
    Confirm the server is accepting connections.

    ```bash
    curl -i http://localhost:8080
    ```

    Any HTTP response (even a 404 from an unmatched route) confirms the server is running.
  </Step>

  <Step title="Send a booking request">
    Submit a booking with a user ID and a list of show seat IDs.

    <CodeGroup>

    ```bash cURL
    curl --request POST \
      --url http://localhost:8080/tickets/book \
      --header 'Content-Type: application/json' \
      --data '{
        "userId": 1,
        "showSeatIds": [1, 2]
      }'
    ```

    ```json Request body
    {
      "userId": 1,
      "showSeatIds": [1, 2]
    }
    ```

    </CodeGroup>

    A successful response looks like this:

    ```json
    {
      "status": "SUCCESS",
      "ticket": {
        "id": 1,
        "show": { ... },
        "seats": [ { ... }, { ... } ],
        "timeOfBooking": "2026-04-23T10:00:00.000+00:00",
        "user": { ... },
        "status": "UNPAID"
      }
    }
    ```

    If the seats are already reserved or the IDs are invalid, the response returns `"status": "FAILURE"` with a `null` ticket.
  </Step>
</Steps>

## Response structure

The booking endpoint returns a `BookTicketResponseDTO` with two fields:

| Field | Type | Description |
|---|---|---|
| `status` | `ResponseStatus` | `SUCCESS` when all seats were reserved; `FAILURE` otherwise |
| `ticket` | `Ticket` | The created ticket object. `null` on failure |

The `Ticket` object includes the booked `Show`, the list of reserved `Seat` objects, the `User`, the time of booking, and a `TicketStatus` (initially `UNPAID`).

<Note>
  The API uses an H2 in-memory database. All data is reset each time the server restarts. You will need to re-seed any users, shows, and seats before making requests after a restart.
</Note>

<Tip>
  Because `spring-boot-devtools` is included in `pom.xml`, the H2 web console is enabled automatically in development mode. Access it at [http://localhost:8080/h2-console](http://localhost:8080/h2-console) and log in with:

  - **JDBC URL**: `jdbc:h2:mem:bms`
  - **Username**: `sa`
  - **Password**: `password`
</Tip>
