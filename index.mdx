The Book My Show Ticket Booking API is a Spring Boot service that lets you book movie tickets programmatically. It handles seat availability checks, atomic reservations (all seats succeed or none do), and tracks each ticket through its full lifecycle from `UNPAID` to `PAID`.

## How it works

The API enforces a strict seat reservation lifecycle to prevent double-booking under concurrent load:

<Steps>
  <Step title="Submit a booking request">
    Call `POST /tickets/book` with your `userId` and a list of `showSeatIds`. The API validates that the user and all requested seats exist.
  </Step>
  <Step title="Seats are atomically reserved">
    The service checks every seat's status. If all are `AVAILABLE`, they are transitioned to `BLOCKED` and a `Ticket` is created with status `UNPAID`. If even one seat is unavailable, the entire request is rejected.
  </Step>
  <Step title="Proceed to payment">
    After receiving the ticket, initiate payment. On success, seat statuses advance to `BOOKED` and the ticket becomes `PAID`.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="All-or-nothing booking" icon="shield-check">
    If any requested seat is unavailable, no seats are reserved and the exact list of conflicting seat IDs is returned.
  </Card>
  <Card title="Concurrent request safety" icon="bolt">
    Designed to handle high-volume, simultaneous booking requests without race conditions.
  </Card>
  <Card title="Full seat status lifecycle" icon="arrow-right-arrow-left">
    Seats flow through `AVAILABLE → BLOCKED → BOOKED` with clear transitions at each stage.
  </Card>
  <Card title="In-memory H2 database" icon="server">
    Uses an embedded H2 database for zero-configuration local development and testing.
  </Card>
</CardGroup>
